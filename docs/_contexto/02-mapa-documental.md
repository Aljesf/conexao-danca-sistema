# 📘 Mapa Documental do Projeto — Leitura Guiada

## 1. Objetivo

Este documento orienta a leitura dos arquivos principais do projeto, indicando quais documentos são:

- normativos
- operacionais
- estruturais
- históricos
- de contexto rápido

## 2. Grupos principais de documentos

### 2.1 Contexto rápido

Pasta:

- `docs/_contexto/`

Função:

- leitura rápida para agentes e novos chats

### 2.2 Normas e instruções de trabalho

Exemplos:

- `docs/instrucoes-oficiais-chatgpt-2.0.md`
- `docs/instrucao_rastreabilidade.md`

Função:

- governança do fluxo GPT × Codex
- rastreabilidade
- organização das entregas

### 2.3 Modelos estruturais

Exemplos:

- `docs/modelo-banco-padrao.md`
- `docs/modelo_financeiro.md`
- `docs/Modelo de Turmas — Conexão Dança 1.3.md`

Função:

- definir estrutura conceitual do sistema

### 2.4 Estado atual / fotografia real

Exemplos:

- `docs/estado-atual-do-projeto.md`
- outros arquivos de estado atual

Função:

- registrar o que já está implementado ou validado

### 2.5 Documentação por domínio

Exemplos:

- matrículas
- financeiro
- documentos
- loja
- café
- movimento
- diário de classe

Função:

- aprofundamento por módulo

## 3. Ordem recomendada de leitura para agentes

1. `docs/_contexto/README.md`
2. `docs/_contexto/00-visao-geral.md`
3. `docs/_contexto/01-nomenclatura-oficial.md`
4. `docs/estado-atual-do-projeto.md`
5. documentos do domínio específico da tarefa

## 4. Regra prática

Antes de qualquer alteração no sistema, o agente deve entender:

- o nome oficial de negócio
- o estado atual real
- o domínio específico da tarefa
